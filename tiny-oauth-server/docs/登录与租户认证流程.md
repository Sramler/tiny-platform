# 登录与租户认证流程（权威版）

最后更新：2026-02-10
适用模块：`tiny-oauth-server`

## 1. 文档目标

本文档是当前登录链路的唯一权威说明，覆盖：

- 租户参数如何进入并贯穿登录流程
- `security.mfa.mode`（`NONE`/`OPTIONAL`/`REQUIRED`）在登录与授权阶段的实际行为
- TOTP 绑定/验证/跳过的真实分支
- Access Token / ID Token 中 `tenantId`、`amr` 的产生规则

## 2. 核心结论（先看这个）

1. `/login` 只是表单认证入口，本身不直接返回 OAuth Token。
2. 登录表单必须带 `tenantId`，否则会被 `TenantContextFilter` 拒绝（`400 missing_tenant`）。
3. 是否要求 TOTP 由后端 `requireTotp` 决策，不由前端页面决定。
4. Access Token / ID Token 中 `tenantId` 已实现，`amr` 已实现。
5. `mode=NONE` 下不应出现 `totp`；出现说明会话被错误升级或走了不符合预期的路径。

## 3. 端到端时序

### 3.1 表单登录阶段

1. 前端 `Login.vue` 提交 `POST /login`，参数包含：
   - `tenantId`
   - `username`
   - `password`
   - `authenticationProvider=LOCAL`
   - `authenticationType=PASSWORD`
2. `TenantContextFilter` 解析租户：
   - 优先已认证上下文（Session/JWT）
   - 其次请求头 `X-Tenant-Id`
   - 登录场景兜底读取 `/login` 的表单参数 `tenantId`
3. `DefaultSecurityConfig` 的 `formLogin().loginProcessingUrl("/login")` 进入认证链。
4. `MultiAuthenticationProvider` 校验用户名密码，并基于 `SecurityService.getSecurityStatus()` 计算 `requireTotp`。

### 3.2 登录成功后的分流（`CustomLoginSuccessHandler`）

`CustomLoginSuccessHandler` 决定跳转到哪里：

1. `disableMfa=true`（`mode=NONE`）
   - 直接跳转业务页
   - **不会**人为补充 TOTP 因子
2. 已完成 `PASSWORD` 但未完成 `TOTP`，且 `requireTotp=true`
   - 跳转 `/self/security/totp-verify?redirect=...`
3. `totpBound=true` 但 `totpActivated=false`
   - 跳转 `/self/security/totp-bind?redirect=...`
4. 未绑定 TOTP 且（强制 MFA 或未跳过提醒）
   - 跳转 `/self/security/totp-bind?redirect=...`
5. 其余情况
   - 直接进入业务页
   - 当 `requireTotp=true` 时，升级为完全认证会话

### 3.3 OIDC 授权与发码阶段

前端通过 OIDC 重定向流程访问 `/oauth2/authorize` 后，`MfaAuthorizationEndpointFilter` 再次兜底：

- `forceMfa=true` 且未绑定/未激活：强制去绑定页
- `requireTotp=false`：放行授权端点
- `requireTotp=true` 且当前认证未完成 TOTP：强制去验证页
- 仅当本次所需因子都完成时才放行授权码签发

这一步确保最终 token 的 `amr` 与真实认证因子一致。

## 4. MFA 模式矩阵

| 模式 | `requireTotp` 计算规则 | 未绑定用户 | 已绑定并激活用户 | `amr` 预期 |
|---|---|---|---|---|
| `NONE` | 恒为 `false` | 直接登录 | 直接登录（不要求 TOTP） | 通常仅 `password` |
| `OPTIONAL` | 仅当已绑定且激活时为 `true` | 引导绑定页，可跳过 | 必须做 TOTP 校验 | `password, totp` |
| `REQUIRED` | 已绑定且激活时为 `true`；未绑定由强制流程拦截 | 必须绑定，不可跳过 | 必须做 TOTP 校验 | `password, totp` |

说明：`skipMfaRemind` 当前尚未持久化到数据库（`SecurityServiceImpl` 内有 TODO），仅用于当前逻辑分支控制。

## 5. 租户传递与隔离

### 5.1 登录前

- 前端 `Login.vue` 提交时会把租户写入：
  - 表单字段 `tenantId`
  - 浏览器存储（`app_tenant_id`）

### 5.2 登录后

- 前端 OIDC / API 请求会在 header 带 `X-Tenant-Id`
- 后端 `TenantContextFilter` 会校验：
  - token/session 中租户与 header 租户不一致时，返回 `403 tenant_mismatch`

### 5.3 CORS

`CorsConfig` 已允许以下请求头（含大小写变体）：

- `X-Tenant-Id`
- `x-tenant-id`

如果浏览器仍报预检失败，先确认服务是否加载到最新代码和配置。

## 6. Token Claims 现状

由 `JwtTokenCustomizer` 添加：

- `userId`
- `username`
- `tenantId`
- `authorities`
- `client_id`
- `scope`
- `auth_time`
- `amr`

`amr` 生成规则：

- 来自 `MultiFactorAuthenticationToken.completedFactors`
- `PASSWORD -> password`
- `TOTP -> totp`
- 若无法解析且已认证，兜底 `password`

### 7.1 页面路由

- `/login`

### 7.2 后端接口

- `POST /oauth2/token`

## 8. 问题排查顺序

1. 先看 `TenantContextFilter` 是否命中 `missing_tenant` / `tenant_mismatch`。
2. 再看 `MultiAuthenticationProvider` 日志中的 `requireTotpThisSession`。
3. 再看 `CustomLoginSuccessHandler` 的分支日志（是否跳到 bind/verify）。
4. 最后看 `MfaAuthorizationEndpointFilter` 是否在 `/oauth2/authorize` 再次拦截。
5. 用 OIDC Debug 页解码 token，核对 `tenantId` 和 `amr`。
